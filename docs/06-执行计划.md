## P0：项目脚手架 + 基础数据管理（2周）

### Day 1-2：项目初始化

**后端（.NET 10）**
```
MES-Framework/src/
├── MES.Api/                  # WebAPI 项目
│   ├── Controllers/
│   ├── Middleware/            # 异常处理、JWT、请求日志
│   └── Program.cs
├── MES.Application/          # 应用服务层
│   ├── Services/             # 业务服务
│   ├── Dtos/                 # 数据传输对象
│   └── Interfaces/           # 服务接口
├── MES.Domain/               # 领域层
│   ├── Entities/             # 实体
│   ├── Enums/                # 枚举（工单状态、质检结果等）
│   └── ValueObjects/
├── MES.Infrastructure/       # 基础设施层
│   ├── Data/                 # EF Core DbContext + 仓储实现
│   ├── Repositories/         # 仓储接口
│   └── Extensions/
└── MES.sln
```

**前端（Vue 3 + Element Plus + TypeScript）**
```
MES-Framework/src/
├── mes-web/                   # 前端项目
│   ├── src/
│   │   ├── api/              # Axios 封装 + API 模块
│   │   ├── views/            # 页面
│   │   ├── components/       # 公共组件
│   │   ├── router/           # 路由
│   │   ├── stores/           # Pinia 状态管理
│   │   └── layouts/          # 布局
│   ├── package.json
│   └── vite.config.ts
```

## 优先级建议

| 阶段 | 之后可以独立交付 | 价值 |
|------|---------|------|
| **P0** | ✅ 基础数据管理功能 | 后台管理可用 |
| **P0+P1** | ✅ 工单全流程可用 | 核心 MVP 可试跑 |
| **P0+P1+P2** | ✅ 现场可用的 MES | 第一次可正式上线 |
| P3-P6 | 按需叠加 | 锦上添花 |

**建议冲刺节奏**：P0+P1+P2 共 6 周出 MVP，够了就可以拿给现场试用，再边用边加 P3-P6。

## 开始条件

1. 安装 .NET 8 SDK（`dotnet --version` 确认）
2. 安装 Node.js 18+（`node -v` 确认）
3. 本地 PostgreSQL 或 Docker
4. Git 仓库初始化
