### Task #2: 修复 AndonService - 数据库持久化

**严重性:** 🔴 严重 - 重启丢失所有安灯事件

**问题描述:**
- `AndonService` 使用 `private readonly List<AndonEvent> _events = new()` 内存存储
- 无 `AndonEvent` 实体、DbSet、数据库表
- 重启应用后所有历史记录丢失

**修复方案:**
```
1. 创建 AndonEvent 实体 (MES.Domain/Entities/AndonEvent.cs)
   - Id, EventType, Level, Title, Message, RelatedEntityType, RelatedEntityId
   - TriggeredBy, TriggeredAt, ResolvedAt, ResolvedBy, IsResolved

2. 添加数据库迁移/更新 init.sql
   - CREATE TABLE mes_andon_events (...)

3. 在 MesDbContext 添加 DbSet<AndonEvent>

4. 重构 AndonService
   - 注入 IRepository<AndonEvent>
   - 方法: GetActiveEventsAsync, GetAllEventsAsync, TriggerAsync, ResolveAsync
   - 保留 SignalR 实时推送

5. 添加单元测试
```

**涉及文件:**
- `src/MES.Domain/Entities/AndonEvent.cs` (新建)
- `src/MES.Infrastructure/Data/MesDbContext.cs`
- `src/MES.Application/Services/AndonService.cs`
- `src/MES.Api/Controllers/AndonController.cs` (修复空 catch 块)
- `database/init.sql`

**依赖:** 无

### Task #3: 修复前后端 API - Integration 模块

**严重性:** 🟡 高 - 前后端接口严重不匹配

**问题描述:**
| 前端调用 | 后端实际 |
|----------|----------|
| `GET /integration/adapters` | `GET /integration/adapters/status` |
| `POST /integration/adapters/{name}/test` | `GET /integration/erp/test`, `GET /integration/wms/test` |
| `POST /integration/adapters/{name}/sync` | 多个细粒度接口 |
| `GET /integration/events` | `GET /integration/event-logs` |

**修复方案 (二选一):**

**方案 A: 以后端为准 - 调整前端**
```
1. 修改 src/mes-web/src/api/integration.ts
   - getAdapters() → getAdapterStatus()
   - testAdapter(name) → testErp() / testWms() / testPlc() (固定名称)
   - syncAdapter(name) → 改为调用具体细粒度接口
   - getLogs() → getEventLogs()

2. 修改 IntegrationDashboard.vue 和 IntegrationLogs.vue
   - 适配新的 API 调用
```

**方案 B: 以后端为准 - 扩展后端**
```
1. 添加 GET /integration/adapters (返回适配器列表)
2. 添加 POST /integration/adapters/{name}/test
3. 添加 POST /integration/adapters/{name}/sync
4. 添加 GET /integration/events (alias of event-logs)

推荐方案 B，保持前端简洁
```

**涉及文件:**
- `src/MES.Api/Controllers/IntegrationController.cs`
- `src/mes-web/src/api/integration.ts`
- `src/mes-web/src/views/integration/dashboard/IntegrationDashboard.vue`
