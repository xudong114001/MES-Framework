## 6. 性能测试（k6）

### 6.1 场景设计

| 场景 | 模拟用户 | 每秒请求 | 持续时间 | 目标 |
|------|---------|---------|---------|------|
| 报工高并发 | 50 并发 | 逐步加到 100 TPS | 5 min | P95 ≤ 500ms |
| 列表查询 | 20 并发 | 50 QPS | 5 min | P95 ≤ 1s |
| 混合负载 | 工单(70%)+报工(20%)+查询(10%) | 100 RPS | 10 min | 无超时错误 |
| 峰值冲击 | 瞬时 200 并发 | 200 RPS | 30s | 不崩溃，错误率 < 1% |

### 6.2 关键指标

| 指标 | 目标值 |
|------|-------|
| P95 响应时间 | ≤ 500ms（报工接口 ≤ 300ms） |
| 错误率 | < 0.1% |
| TPS | ≥ 100 transactions/s |
| CPU 使用率 | < 70% |
| 内存 | < 2GB（单实例） |
| DB 连接池 | < 80% 利用率 |

### 6.3 测试脚本示例

```javascript
// k6 报工接口压测脚本示例
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '1m', target: 20 },   // 热身
    { duration: '3m', target: 100 },  // 爬升
    { duration: '3m', target: 100 },  // 峰值保持
    { duration: '1m', target: 0 },    // 降冷
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'],
    http_req_failed: ['rate<0.001'],
  },
};

export default function () {
  const payload = JSON.stringify({
    work_order_id: 300001,
    step_id: 400001,
    workstation_id: 500001,
    operator_id: 'USER001',
    report_type: 'COMPLETE',
    good_qty: 10,
    scrap_qty: 0,
    rework_qty: 0,
    duration_min: 5,
  });

  const res = http.post('http://localhost:5180/api/v1/work-reports', payload, {
    headers: { 'Content-Type': 'application/json' },
  });

  check(res, { 'status is 200': (r) => r.status === 200 });
  sleep(0.5);
}
```

## 7. 验收测试（端到端）

### 7.1 核心流程冒烟测试

```
场景：完整的生产批次

1. 登录系统
2. 创建物料 A（成品）和物料 B、C（子件）
3. 维护 BOM：A = B×2 + C×1
4. 维护工艺路线：A 的工艺 = 工序10(上料) → 工序20(组装) → 工序30(测试)
5. 创建工单：生产 A × 100
6. 下达工单 → 系统生成三个工序任务
7. 排产到产线 1
8. 工序10 报工：good=100 → 工序10 完成
9. 工序20 报工：good=95, scrap=3, rework=2 → 工序20 完成
10. 工序30 质检：process_qc=90, scrap=5 → 首次质检不合格
11. 返工 5 件 → 再次报好工
12. 关闭工单 → 工单 COMPLETED
13. 按批次号追溯：原料 B、C 的去向
14. 按序列号追溯：成品 A 的加工记录
```

### 7.2 异常场景

No. | 场景 | 预期
----|------|------
E2E-01 | 网络断开后重新连接 | 已报工数据不丢失
E2E-02 | 扫码枪输入乱码 | 系统容错处理
E2E-03 | 同时两个终端报同一工单 | 总数量不超过计划
E2E-04 | 中途取消工单后恢复 | 不可恢复，需重新建单
E2E-05 | 导入 10 万条物料数据 | 速度可接受，页面不崩
