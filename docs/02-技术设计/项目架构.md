## 3. 核心设计模式

### 3.1 工单状态机

```
PENDING → RELEASED → SCHEDULED → IN_PROGRESS → COMPLETED → CLOSED
  ↓          ↓           ↓              ↓
CANCELLED  CANCELLED  CANCELLED      ON_HOLD → IN_PROGRESS
                                      ↓
                                   CANCELLED
```

状态转换校验：每种状态只能按预定义的方向流转，非法操作抛异常。

### 3.2 领域事件驱动

```
工单完工 → 触发 QualityCheckNeeded 事件 → 创建质检任务
工单完工 → 触发 MaterialConsume 事件    → 扣减物料库存
质检完成 → 触发 InventoryUpdate 事件    → 更新成品库存
报工完成 → 触发 ProductionOutput 事件   → 推送看板
```

## 5. 扩展性设计

- **多租户**: 按工厂 tenant_id 隔离数据（推荐共享数据库 + 行级隔离）
- **插件化**: 业务逻辑预留扩展点（如报工前校验接口、工单审批链）
- **外部集成**: 通过消息队列解耦与 ERP/WMS/SCADA 的数据交换

### 6.4 预警规则说明

| 规则 | 触发条件 | 预警级别 |
|------|---------|----------|
| 产线连续不良率飙升 | 连续3个工单报废率 > 5% | High |
| 物料批次异常 | 某批次物料在≥2个工单不良率>3% | Critical |
| 工位连续返工 | 同一工位连续≥5次返工 | Medium |

### 6.4 设备健康度评分

| 健康度 | 风险等级 | 建议措施 |
|--------|---------|----------|
| 90-100 | 优 | 常规监控 |
| 70-89 | 良 | 关注趋势变化 |
| 50-69 | 预警 | 近期安排维护 |
| <50 | 危险 | 立即停机检修 |

### 6.5 与现有架构的集成

| 现有模块 | AI 调用点 | 数据流向 |
|----------|-----------|----------|
| WorkReportService | 报工完成后 → 触发质量分析 | AI 读取报工数据，返回预警 |
| SchedulingService | 排程时 → 请求 AI 建议 | AI 返回推荐产线+置信度 |
| EquipmentService | 定时任务 → OEE 趋势分析 | AI 返回健康度评分+维护预测 |

> **注**: AI 模块以新增项目方式接入（MES.AI.Application），零修改现有业务代码。

## 8. RBAC 权限管理

### 8.1 数据模型

```
MES.Domain/
├── Entities/
│   ├── Role.cs              # 角色实体 (Id, Name, Description)
│   └── UserRole.cs          # 用户角色关联 (UserId, RoleId)
```

### 8.2 授权策略

| 角色 | 可访问模块 |
|------|-----------|
| admin | 全部权限 |
| supervisor | 基础数据、工单、报工、排产、设备 |
| operator | 工单查看、报工、PDA |
| inspector | 质量管理、追溯查询 |
| viewer | 看板、报表查看 |

JWT 登录时从数据库加载用户角色并注入 Claims，Controller 层通过 `[Authorize(Roles = "...")]` 声明式控制。

## 7. 集成适配器架构

### 7.1 适配器模式

系统采用适配器模式集成外部系统，通过 `IERPAdapter`、`IWMSAdapter`、`IPlcCollector` 接口统一抽象：

```
MES.Integration/
├── Adapters/
│   ├── IERPAdapter.cs           # ERP 适配器接口
│   ├── IWMSAdapter.cs           # WMS 适配器接口
│   ├── IPlcCollector.cs         # PLC 采集器接口
│   ├── MockERPAdapter.cs        # Mock ERP（测试用）
│   ├── MockWMSAdapter.cs        # Mock WMS
│   ├── MockPlcCollector.cs      # Mock PLC
│   ├── SapB1Adapter.cs          # SAP Business One 适配器
│   ├── KingdeeAdapter.cs        # 金��� K3 Cloud 适配器
│   └── ModbusTcpCollector.cs    # Modbus TCP PLC 采集器
├── EventBus/
│   ├── IEventBus.cs             # 事件总线接口
│   ├── RabbitMQEventBus.cs      # RabbitMQ 实现
│   └── SubscriptionManager.cs   # 订阅管理
└── Plc/
    ├── IPlcCollector.cs         # PLC 数据采集接口
    ├── ModbusTcpCollector.cs    # Modbus TCP 协议实现
    └── MockPlcCollector.cs      # Mock 实现
```

### 7.2 适配器类型

| 类型 | 已实现适配器 | 配置项 |
|------|-------------|--------|
| ERP | MockERP / SAP B1 / 金蝶 | `Integration:ERP:Type` |
| WMS | MockWMS | `Integration:WMS:Type` |
| PLC | MockPlcCollector / ModbusTcpCollector | `Integration:Plc:Type` |

### 7.3 DI 注册机制

适配器通过配置文件动态注册：

```json
{
  "Integration": {
    "ERP": { "Type": "SapB1", "SapB1": { ... } },
    "Plc": { "Type": "Modbus", "Modbus": { ... } }
  }
}
```
