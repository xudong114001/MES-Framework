## 3. 核心设计模式

### 3.1 工单状态机

```
PENDING → RELEASED → SCHEDULED → IN_PROGRESS → COMPLETED → CLOSED
  ↓          ↓           ↓              ↓
CANCELLED  CANCELLED  CANCELLED      ON_HOLD → IN_PROGRESS
                                      ↓
                                   CANCELLED
```

状态转换校验：每种状态只能按预定义的方向流转，非法操作抛异常。

### 3.2 领域事件驱动

```
工单完工 → 触发 QualityCheckNeeded 事件 → 创建质检任务
工单完工 → 触发 MaterialConsume 事件    → 扣减物料库存
质检完成 → 触发 InventoryUpdate 事件    → 更新成品库存
报工完成 → 触发 ProductionOutput 事件   → 推送看板
```

## 5. 扩展性设计

- **多租户**: 按工厂 tenant_id 隔离数据（推荐共享数据库 + 行级隔离）
- **插件化**: 业务逻辑预留扩展点（如报工前校验接口、工单审批链）
- **外部集成**: 通过消息队列解耦与 ERP/WMS/SCADA 的数据交换

## 6. AI 智能模块

### 6.1 模块架构

```
MES.AI.Domain/                    # AI 领域模型
├── Entities/
│   ├── AlertRecord.cs           # 预警记录
│   └── AlertRule.cs             # 预警规则定义
└── Enums/
    └── AlertLevel.cs            # 预警级别(低/中/高/严重)

MES.AI.Application/               # AI 服务层
├── Services/
│   ├── QualityAlertService.cs       # 质量预警引擎
│   ├── SchedulingRecommendationService.cs  # 智能排程建议
│   └── EquipmentHealthService.cs    # 设备预测维护
├── Dtos/
│   ├── ScheduleRecommendationDto.cs
│   └── EquipmentHealthDto.cs
└── Interfaces/
    ├── IQualityAlertService.cs
    ├── ISchedulingRecommendationService.cs
    └── IEquipmentHealthService.cs
```

### 6.2 AI 功能矩阵

| 功能 | 技术方案 | 数据来源 | 状态 |
|------|---------|---------|------|
| 质量异常预警 | 规则引擎（3条规则） | WorkReport, MaterialTrace | ✅ 已实现 |
| 智能排程建议 | 多因子加权算法 | WorkOrder, ProductionLine, Routing | ✅ 已实现 |
| 设备预测维护 | OEE趋势分析+线性回归 | WorkReport, Equipment OEE | ✅ 已实现 |

### 6.3 预警规则说明

| 规则 | 触发条件 | 预警级别 |
|------|---------|----------|
| 产线连续不良率飙升 | 连续3个工单报废率 > 5% | High |
| 物料批次异常 | 某批次物料在≥2个工单不良率>3% | Critical |
| 工位连续返工 | 同一工位连续≥5次返工 | Medium |

### 6.4 设备健康度评分

| 健康度 | 风险等级 | 建议措施 |
|--------|---------|----------|
| 90-100 | 优 | 常规监控 |
| 70-89 | 良 | 关注趋势变化 |
| 50-69 | 预警 | 近期安排维护 |
| <50 | 危险 | 立即停机检修 |

### 6.5 与现有架构的集成

| 现有模块 | AI 调用点 | 数据流向 |
|----------|-----------|----------|
| WorkReportService | 报工完成后 → 触发质量分析 | AI 读取报工数据，返回预警 |
| SchedulingService | 排程时 → 请求 AI 建议 | AI 返回推荐产线+置信度 |
| EquipmentService | 定时任务 → OEE 趋势分析 | AI 返回健康度评分+维护预测 |

> **注**: AI 模块以新增项目方式接入（MES.AI.Application），零修改现有业务代码。
